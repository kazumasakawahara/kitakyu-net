# スプレッドシートテンプレート説明

## 概要
このファイルは、Googleスプレッドシートで事業所情報を管理するためのテンプレート構造を説明します。

## シート構成

### 1. basic_info（基本情報）

必須項目を中心とした事業所の基本情報を管理します。

| 列名 | データ型 | 必須 | 説明 | 例 |
|------|---------|------|------|-----|
| 事業所ID | テキスト | ○ | UUID形式の一意識別子 | abc123-def456-... |
| 事業所名 | テキスト | ○ | 正式な事業所名 | ○○作業所 |
| 法人名 | テキスト | ○ | 運営法人名 | 社会福祉法人△△会 |
| 事業所番号 | テキスト | ○ | 10桁の指定番号 | 4010100123 |
| サービス種別 | テキスト | ○ | サービスの種類 | 就労継続支援B型 |
| サービスカテゴリ | テキスト | ○ | 介護給付/訓練等給付 | 訓練等給付 |
| 郵便番号 | テキスト | - | 000-0000形式 | 802-0001 |
| 住所 | テキスト | ○ | 完全な住所 | 福岡県北九州市小倉北区○○1-2-3 |
| 所在区 | テキスト | ○ | 北九州市の区 | 小倉北区 |
| 電話番号 | テキスト | ○ | 0000-00-0000形式 | 093-123-4567 |
| FAX | テキスト | - | FAX番号 | 093-123-4568 |
| メール | テキスト | - | メールアドレス | info@example.com |
| 定員 | 数値 | - | 利用定員 | 20 |
| 空き状況 | 選択 | - | 空きあり/要確認/満員等 | 空きあり |
| 営業日 | テキスト | - | 営業日の説明 | 月〜金 |
| 営業時間 | テキスト | - | 営業時間の説明 | 9:00〜16:00 |
| データソース | テキスト | ○ | データの出所 | WAM_NET |
| 最終更新日 | 日付 | ○ | 最終更新日時 | 2025-10-31 |
| 更新者 | テキスト | - | 更新者名 | Kazumasa |

### 2. service_details（サービス詳細）

サービス提供体制や対応可能な範囲を管理します。

| 列名 | データ型 | 必須 | 説明 | 例 |
|------|---------|------|------|-----|
| 事業所ID | テキスト | ○ | basic_infoと紐付け | abc123-def456-... |
| 対応障害種別 | テキスト | - | カンマ区切りで複数可 | 知的障害,精神障害 |
| 対応支援区分 | テキスト | - | カンマ区切りで複数可 | 区分3,区分4,区分5 |
| 送迎の有無 | TRUE/FALSE | - | 送迎サービスの有無 | TRUE |
| 送迎エリア | テキスト | - | 送迎可能エリア | 小倉北区全域 |
| 食事提供 | TRUE/FALSE | - | 食事提供の有無 | TRUE |
| 医療的ケア対応 | TRUE/FALSE | - | 医療的ケアの可否 | FALSE |
| 強度行動障害対応 | TRUE/FALSE | - | 強度行動障害への対応 | FALSE |
| 主な活動内容 | テキスト | - | メインの活動内容 | 軽作業・創作活動 |
| 作業種目 | テキスト | - | 具体的な作業内容 | 箱折り,クリーニング |
| 工賃情報 | テキスト | - | 平均工賃等 | 月平均15,000円 |

相談支援事業所との連携実績や事業所の実績を管理します。

| 列名 | データ型 | 必須 | 説明 | 例 |
|------|---------|------|------|-----|
| 事業所ID | テキスト | ○ | basic_infoと紐付け | abc123-def456-... |
| 相談支援連携実績 | TRUE/FALSE | - | 連携実績の有無 | TRUE |
| サービス管理責任者名 | テキスト | - | サビ管の氏名 | 山田太郎 |
| 管理者名 | テキスト | - | 管理者の氏名 | 鈴木花子 |
| 連携時の特記事項 | テキスト | - | 連携上の注意点等 | 月初に連絡が取りやすい |
| 地域移行実績 | テキスト | - | 施設からの移行実績 | 過去3年で2名 |
| 就労移行実績 | テキスト | - | 一般就労への移行実績 | 過去3年で5名 |

### 4. individual_notes（個別メモ）

利用者ごとの個別的な情報を管理します。（個人情報注意）

| 列名 | データ型 | 必須 | 説明 | 例 |
|------|---------|------|------|-----|
| 事業所ID | テキスト | ○ | basic_infoと紐付け | abc123-def456-... |
| 利用者ID | テキスト | - | 利用者の識別子（匿名化） | USER001 |
| 本人との相性 | テキスト | - | 本人の反応・評価 | 活動内容が合っている |
| 家族の評価 | テキスト | - | 家族の感想・評価 | 満足している |
| 支援上の注意点 | テキスト | - | 支援時の留意事項 | 午前中の集中力が高い |
| 最終連絡日 | 日付 | - | 最後に連絡した日 | 2025-10-15 |
| 次回連絡予定 | 日付 | - | 次回連絡予定日 | 2025-11-15 |

### 5. master_data（マスターデータ）

ドロップダウンリストや参照用のマスターデータを管理します。

| 列名 | データ型 | 必須 | 説明 |
|------|---------|------|------|
| カテゴリ | テキスト | ○ | データのカテゴリ（サービス種別、障害種別等） |
| 値 | テキスト | ○ | 実際の値 |
| 説明 | テキスト | - | 値の説明 |

**マスターデータの例:**
```
カテゴリ: サービス種別
値: 就労継続支援B型
説明: 雇用契約を結ばずに就労の機会を提供

カテゴリ: 障害種別
値: 知的障害
説明: 療育手帳を所持している方

カテゴリ: 所在区
値: 小倉北区
説明: 北九州市小倉北区
```

## データ入力のルール

### 1. 必須項目
- 事業所ID（自動生成推奨）
- 事業所名
- 法人名
- 事業所番号
- サービス種別
- 電話番号
- 住所

### 2. データ形式
- 電話番号: 093-123-4567 形式
- 郵便番号: 802-0001 形式
- 日付: YYYY-MM-DD形式
- TRUE/FALSE: 大文字で統一
- 複数値: カンマ区切り（スペースなし）

### 3. 空き状況の選択肢
- 空きあり
- 要確認
- 待機中
- 満員
- 不明

### 4. データ検証
- 事業所番号は10桁の数字
- 電話番号は正しい形式
- 郵便番号は正しい形式
- 所在区は北九州市の7区のいずれか

## 実装時の注意点

1. **Googleスプレッドシートでの実装:**
   - 各列にデータ検証を設定（ドロップダウン等）
   - 条件付き書式で必須項目を強調
   - シート保護で誤編集を防止

2. **データインポート時:**
   - 事業所IDで重複チェック
   - 必須項目の入力チェック
   - データ型の検証

3. **個人情報の取り扱い:**
   - individual_notesシートは特に厳重管理
   - 利用者IDは匿名化
   - 定期的なアクセス権限の確認
